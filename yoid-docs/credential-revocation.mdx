---
title: "Credential Revocation"
description: "Revoke credentials that are no longer valid."
---

## Overview

Credentials in the YoID ecosystem can be revoked when they are no longer valid. This is a trust story as much as a technical one — a training partner who discovers a fraudulent completion, or a programme that expires credentials annually, needs the ability to invalidate previously issued credentials.

When a credential is revoked, any future verification attempt against it will show `isValid: false`.

## When to Revoke

| Scenario | Example |
|----------|---------|
| **Fraudulent issuance** | A learning partner discovers a youth didn't actually complete the programme |
| **Credential expiry** | Annual certifications that need to be renewed |
| **Data correction** | A credential was issued with incorrect attributes and needs to be re-issued |
| **Programme withdrawal** | A youth withdraws from a programme after a credential was issued |

## Endpoint

```
POST https://test.didxtech.com/me-creds/api/credentials/{id}/revoke
```

## Request

```bash
curl -X POST "https://test.didxtech.com/me-creds/api/credentials/iss_cm7d9cq5600pwk3zq3kmpg8uc/revoke" \
  -H "Authorization: Bearer <access_token>"
```

Replace the ID with the issuance `id` returned when the credential was originally issued (Step 3 of the issuer workflow).

## Response

**HTTP 200 OK**

The credential is marked as revoked. Future verification attempts against this credential will return `isValid: false` with an issue indicating revocation.

## Impact on Verification

When a verifier (like JobJack) checks a credential that has been revoked:

```json
{
  "credentials": [
    {
      "isValid": false,
      "name": "Web Development Completion",
      "type": "https://metadata.paradym.id/types/web-dev-completion-v1",
      "attributes": { ... },
      "issues": ["Credential has been revoked by the issuer"]
    }
  ]
}
```

The verifier's application should check `isValid` and the `issues` array to determine why verification failed.

<Note>
  Revocation is permanent. If you need to re-credential a youth after revoking, issue a new credential through the standard issuance workflow.
</Note>

## Trust Implications

Revocation is part of what makes the ecosystem trustworthy. An employer verifying a credential can be confident that:

1. The credential was genuinely issued by the claimed partner
2. It hasn't been tampered with
3. **It hasn't been revoked** — the issuer still stands behind it

Without revocation, the ecosystem can't handle corrections or fraud. With it, every partner can trust that verified credentials are current and valid.
