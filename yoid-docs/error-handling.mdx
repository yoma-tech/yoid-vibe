## Error Response Format

All APIs return structured JSON error responses:

```json
{
  "error": "VALIDATION_ERROR",
  "message": "The request contains invalid parameters",
  "details": {
    "field": "email",
    "issue": "Must be a valid email address"
  }
}
```

| Field | Description |
|-------|-------------|
| `error` | Machine-readable error code |
| `message` | Human-readable description |
| `details` | Additional context about which field or parameter caused the issue |

## HTTP Status Codes

| Status | Code | Description |
|--------|------|-------------|
| OK | `200` | Request succeeded |
| Created | `201` | Resource created successfully |
| Bad Request | `400` | Invalid or malformed parameters |
| Unauthorized | `401` | Authentication token is missing or expired |
| Not Found | `404` | The requested resource doesn't exist |
| Internal Server Error | `500` | An unexpected server error — contact the YoID team |

## Common Errors

<AccordionGroup>
  <Accordion title="401 Unauthorized — Token Expired">
    The most common error. Your access token expires after **300 seconds** (5 minutes) with no refresh token support.

    **Fix:** Request a new token using the [token caching pattern](/api/authentication#token-caching-pattern). If you see frequent 401s, your token caching isn't refreshing proactively enough — refresh at least 30 seconds before expiry.

    ```json
    {
      "error": "UNAUTHORIZED",
      "message": "Access token is missing or expired"
    }
    ```
  </Accordion>

  <Accordion title="400 Bad Request — Duplicate Email">
    Common in the YOMA ecosystem where youth interact with multiple partners. If Thandi was already onboarded by Umuzi, JA Worldwide will get this error when trying to create her account.

    **Fix:** The youth account already exists. You can proceed to credential issuance — you don't need to create the account again.

    ```json
    {
      "error": "VALIDATION_ERROR",
      "message": "A user with this email already exists"
    }
    ```
  </Accordion>

  <Accordion title="400 Bad Request — Invalid Parameters">
    The request body contains invalid data. Check the `details` field.

    ```json
    {
      "error": "VALIDATION_ERROR",
      "message": "The request contains invalid parameters",
      "details": {
        "field": "credentialTemplateId",
        "issue": "Template not found or has been archived"
      }
    }
    ```
  </Accordion>

  <Accordion title="404 Not Found — Resource Missing">
    A template ID, presentation ID, or credential ID is incorrect or has been deleted.

    **Fix:** Verify the ID is correct and the resource hasn't been archived.

    ```json
    {
      "error": "NOT_FOUND",
      "message": "Credential template not found"
    }
    ```
  </Accordion>

  <Accordion title="500 Internal Server Error">
    An unexpected server error. Not caused by your request. Retry after a short delay.

    **Fix:** If persistent, contact the YoID team at [support@yoma.world](mailto:support@yoma.world).

    ```json
    {
      "error": "INTERNAL_SERVER_ERROR",
      "message": "An unexpected error occurred"
    }
    ```
  </Accordion>
</AccordionGroup>

## Error Handling Pattern

```javascript
async function yoidApiCall(url, options) {
  const token = await getAccessToken(); // Uses caching pattern

  const response = await fetch(url, {
    ...options,
    headers: {
      ...options.headers,
      Authorization: `Bearer ${token}`,
      "Content-Type": "application/json",
    },
  });

  if (response.status === 401) {
    // Token expired — clear cache and retry once
    clearTokenCache();
    const freshToken = await getAccessToken();
    return fetch(url, {
      ...options,
      headers: {
        ...options.headers,
        Authorization: `Bearer ${freshToken}`,
        "Content-Type": "application/json",
      },
    });
  }

  if (!response.ok) {
    const error = await response.json();
    throw new Error(`YoID API error: ${error.error} — ${error.message}`);
  }

  return response;
}
```
